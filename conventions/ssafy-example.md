# 📋 Jira Convention — SSAFY Example

> **예시 문서입니다 — 그대로 쓰지 마세요.** 이 파일은 원작성자(SSAFY 프로젝트) 팀이 사용한 규약이며, 다른 팀의 기본값이 아닙니다. `/setup` 은 기본적으로 `conventions/default.md` (글로벌 표준) 를 사용합니다. 본인 팀 규약이 있다면 `/setup` 의 "팀 컨벤션 있나요?" 질문에서 **yes** 를 선택해 입력하세요.

본 문서는 원작성 팀이 사용한 **Jira 이슈 작성 규칙, Story Point 기준, 작업 흐름**입니다.

---

# 1️⃣ Jira Issue 유형

본 프로젝트에서는 다음 Issue 유형을 사용한다.

| Type | 설명 |
|-----|------|
| Epic | 큰 기능 단위 |
| Story | 사용자 기능 단위 |
| Task | 실제 개발 작업 |
| Bug | 버그 수정 |

일반적인 구조


Epic → Story → Task


---

# 2️⃣ Jira 제목 규칙

Task 제목은 **파트 + 작업 설명** 형식으로 작성한다.
설명은 직관적으로 기재한다.


## 형식


[파트] 작업 설명


### 파트

| 파트 | 설명 |
|-----|------|
| FE | Frontend |
| BE | Backend |
| AI | AI / 모델 |
| DOCS | 문서 |

---

## 제목 예시


[FE] 로그인 페이지 UI 구현
[FE] 감정 드로잉 캔버스 기능 구현

[BE] 로그인 API 구현
[BE] JWT 인증 로직 구현
[BE] 감정 기록 데이터 저장 API 개발

[AI] 그림 감정 분석 모델 연동

[DOCS] Git Convention 문서 작성
[DOCS] API 명세서 작성


---

# 3️⃣ Story Point 기준

작업 난이도와 복잡도를 고려하여 **1 ~ 10 스토리 포인트**로 산정한다.

| Point | 기준 |
|------|------|
| 10 | 매우 위험 / 복잡 — 범위가 너무 커서 반드시 작업을 쪼개야 하는 수준 |
| 9 | 대규모 — 공부해야 할 양이 많고 다른 팀원의 코드와 얽혀 있는 작업 |
| 8 | 고난도 — 로직이 매우 복잡하여 하루 이상 집중해야 할 작업 |
| 7 | 상당한 부담 — 구현해야 할 화면/기능이 많고 예상치 못한 문제가 발생할 가능성이 높은 작업 |
| 6 | 연구 필요 — 처음 사용하는 기술이라 많은 검색과 실험이 필요한 작업 |
| 5 | 표준 작업 — 프로젝트에서 일반적인 기능 개발 수준 |
| 4 | 약간의 고민 — 기존 코드를 참고하면 해결 가능한 작업 |
| 3 | 명확함 — 구현 방법이 명확하고 변수 없이 처리 가능한 작업 |
| 2 | 단순 — 반복 작업 또는 간단한 수정 |
| 1 | 매우 단순 — 오타 수정, 문서 수정 등 |

---

# 4️⃣ Jira 작업 흐름

본 프로젝트는 다음 상태 흐름을 따른다.


To Do → In Progress → Code Review → Done


| 상태 | 설명 |
|-----|------|
| To Do | 작업 예정 |
| In Progress | 작업 진행 중 |
| Code Review | MR 생성 후 코드 리뷰 대기 |
| Done | Merge 완료 |
